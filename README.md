### Задание 1

Опишите основные преимущества использования масштабирования методами:

- активный master-сервер и пассивный репликационный slave-сервер; 
- master-сервер и несколько slave-серверов;


*Дайте ответ в свободной форме.*

---
#### Ответ на задание 1.

1. Активный master‑сервер и пассивный репликационный slave‑сервер

    Такая конфигурация обеспечивает высокую отказоустойчивость, где пассивный slave‑сервер постоянно синхронизируется с master‑сервером и хранит актуальную копию данных. Если основной сервер выходит из строя, то можно оперативно перевести нагрузку на реплику — это минимизирует время простоя.
    Дополнительно slave‑сервер можно задействовать для резервного копирования, например создание бекапов базы данных на реплике не создаёт нагрузки на master‑сервер, что сохраняет производительность основной системы. Это оптимльно для сценариев, где критически важна сохранность данных и быстрая реакция на сбои, но нет высокой нагрузки на чтение.

2. Master‑сервер и несколько slave‑серверов

    Такая конфигурация позволяет масштабировать операции чтения (read scaling) где запросы на чтение данных распределяются между несколькими slave‑серверами, снижая нагрузку на master‑сервер. Он остаётся ответственным только за операции записи (INSERT, UPDATE, DELETE), что повышает общую производительность системы при большом числе запросов на чтение.
    Так же, несколько реплик усиливают отказоустойчивость. Допустим, если один slave‑сервер выходит из строя, то нагрузка перераспределяется между оставшимися. Дополнительно можно географически распределять slave‑серверы, приближая их к пользователям в разных регионах, — это сокращает задержки (latency) при доступе к данным и улучшает пользовательский опыт. Такой подход особенно эффективен для максимально нагруженных веб‑приложений и сервисов с глобальной аудиторией.

- пользователи, 
- книги, 
- магазины (столбцы произвольно).

### Вывод.

Горизонтальный шардинг предполагает разбиение таблицы на две части по количеству строк с размещением на разных серверах — это позволяет равномерно распределить пользовательские запросы и снизить нагрузку на основной сервер.

Вертикальный шардинг выделяет отдельные столбцы или группы данных на отдельные серверы: для таблицы «пользователи» — столбец с паролями, для «книги» — данные о годах написания книг, для «магазины» — площадь магазина. Такой подход оптимизирует скорость ответа при росте числа записей и запросов.
